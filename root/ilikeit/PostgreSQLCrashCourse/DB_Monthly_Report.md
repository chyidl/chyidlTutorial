DB Monthly Report
=================
* Database - 技术方向[云原生数据库]
```
云原生数据库 vs 托管/自建数据库 
  云原生数据库面向独立资源云化，CPU、内存、存储可实现独立弹性
    - 实现存储和计算分离，实现一写 多读集群架构
    - 基于分布式共享存储的一写多读架构，使得数据库的读能力可以快速Scale Out, 同时由于多个读节点和写节点共享同一份存储，降低成本，提升资源利用率
    - AWS - Aurora [云原生]
    - Aliyun - PolarDB 支持最大100T存储，1写15读集群
    - 腾讯 -- CynosDB 
    - 华为云 -- TaurusDB 
    - || 存在瓶颈: 内存弹性， 单点写入 
    - 基于CPU和内存分离的分布式共享内存池 
    -
  托管数据库还局限于传统的服务器架构
```
