DNS 域名解析使用的TCP协议还是UDP协议
------------------------------------
```
DNS同时占用 UDP和TCP 53端口
  1. TCP: 一种面向连接的协议，提供可靠的数据传输, 一般服务质量要求比较高
    TCP 协议中包含专门的传递保证机制，当数据接收方收到发送方传来的信息时，会自动像发送方发送确认信息，发送方只有在就收到该确认信息之后才继续传送其它信息，否则将一直等待直到收到确认信息为止
  2. UDP: 用户数据报协议,是一种无连接的传输层协议, 提供面向事务的简单不可靠信息传输服务
    UDP不能保证数据发送和接收的顺序 
    UDP将安全和排序等功能移交给上层应用来完成，极大降低执行时间，使得速度得到保证

DNS 在进行区域传输的时候使用TCP协议，在其它时候使用UDP协议 
  DNS协议规范2中类型的DNS服务器，一种叫主DNS服务器，一种叫辅助DNS服务器，在一个区中主DNS服务器从自己本机的数据文件中读取DNS数据信息，而辅助DNS服务器则从主DNS服务器中读取该区DNS数据信息，当一个辅助DNS服务器启动时，需要与主DNS服务器通信，并加载数据信息，叫做区传送(zone transfer)

TCP 和 UDP 传送字节的长度限制:
  UDP报文的最大长度为512字节 
  TCP允许报文长度超过512字节 
  当DNS查询超过512字节时， 协议的TC标志出现删除标志，则使用TCP发送 

TCP是一种可靠的连接，保证数据的准确性 
  域名解析时使用UDP协议
    客户端像DNS服务器查询域名，一般返回的内容不会超过512字节，使用UDP传输即可，不用经过TCP三次握手，这样DNS服务器负载更低，响应更快，
```
